## 2단계: 전 5건 통합 구현

### 항목 1+4 (HIGH): _postprocess_remove_fate_mapping 신규 함수

**구현 파일**: `engine/divination/face_reading.py`

1. `_FATE_PATTERNS` — 5카테고리 정규식 목록:
   - `time_fate`: 초년/중년/말년 + 복록/운/길/흉 조합
   - `fate_word`: 대운·금전수·재물수·길흉화복·학문복·재물복·인덕 등
   - `prophecy_ending`: ~될 것이로세, ~의 운이 있다/있도다
   - `school_citation`: 마의상법/신상전편/달마상법
   - `sasang`: 태양인·태음인·소양인·소음인·사상체질·사상의학

2. `_postprocess_remove_fate_mapping(text: str) -> str`:
   - 줄 단위 분리 → 패턴 매치 줄 제거 → 재조합
   - 제거 발생 시 `_FATE_DISCLAIMER` 자동 부착
   - `_call_stage2_persona` Gemini 경로 + Opus fallback 경로 모두 적용

3. `_STAGE2_PERSONA_SYSTEM` 금지어 절에 추가:
   ```
   • 사상체질 인용: 태양인·태음인·소양인·소음인·사상체질 어휘 X
   ```

4. `_STAGE1_OBJECTIVE_SYSTEM` rule 5a 추가:
   ```
   5a. Forbidden traditional Korean medicine / constitution vocabulary:
   태양인, 태음인, 소양인, 소음인, 사상체질, 사상의학.
   ```

### 항목 2 (MEDIUM): HEIC/AVIF 이미지 포맷 감지

**구현 파일**: `engine/divination/face_reading.py`, `engine/safety/file_integrity.py`

`_detect_image_mime` 확장:
- ISO BMFF ftyp box 감지 (bytes 4-7 == `ftyp`)
- brand 분기: heic/heix → `image/heic`, mif1/msf1 → `image/heif`, avif/avis → `image/avif`
- base64 디코딩 후 brand 추출

`file_integrity.py` 동기화:
- `_detect_mime_from_magic` 동일한 ISO BMFF 감지 추가
- HEIC/AVIF 전용 한국어 안내 메시지:
  - HEIC: "HEIC 형식은 지원하지 않습니다. 사진 앱에서 JPG로 내보내기 후 업로드해주세요."
  - AVIF: "AVIF 형식은 지원하지 않습니다. JPG 또는 PNG로 변환 후 업로드해주세요."

### 항목 3 (HIGH): 서버측 5MB 안전망 HTTP 413

**구현 파일**: `web/server.py`

`post_face_reading` 핸들러 앞단에 추가:
```python
_MAX_B64_LEN = 7 * 1024 * 1024  # 7MB base64 ≈ 5MB binary
b64_body = raw_b64.split(",", 1)[-1] if "," in raw_b64 else raw_b64
if len(b64_body) > _MAX_B64_LEN:
    raise HTTPException(status_code=413, detail="이미지가 너무 큽니다 — 5MB 이하 JPG/PNG/WEBP로 변환 후 업로드해주세요.")
```

기존 `file_integrity.py`의 5MB 바이너리 제한과 이중 방어.
서버가 LLM 호출 전 조기 차단으로 502 방지.

### 항목 5 (LOW): 캐시 키 question 제외

**구현 파일**: `engine/divination/face_reading.py`

`_hash_payload` 수정:
- `question` 파라미터 유지 (API 호환) 하지만 해시에 포함하지 않음
- `metrics` 5번째 파라미터 추가 (기존 테스트 4건이 5-arg 방식으로 작성되어 있었음 — 이미 버그였음)
- `generate_face_reading` 내 `_hash_payload` 호출에서 `question=None`, `metrics=metrics` 전달

```python
# 변경 전
key = _hash_payload(image_b64, age, gender, question)
# 변경 후
key = _hash_payload(image_b64, age, gender, None, metrics)
```

효과: 같은 사진/나이/성별이면 question 달라도 캐시 hit → 비용·응답속도 개선.
