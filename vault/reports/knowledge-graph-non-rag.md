# 환각 0% 지식 그래프 + 비-RAG 에이전트 — 사실성 검토

## 보고서 요약

RAG 한계 비판 → Neo4j 지식 그래프 + NL2GQL(Cypher) + 다중 에이전트 (Preprocessor → Generator → Refiner → Formatter) 오케스트레이션 제안.

## 🟢 팩트

| 주장 | 검증 |
|---|---|
| RAG의 환각 패턴 (통계 조작·불완전 검색·도메인 외 날조) | ✅ 인정된 한계 |
| Neo4j Cypher 질의어 | ✅ 표준 |

## 🟡 구조 (시스템 설계 명제)

- 결정론 포매팅 — LLM 생성 능력 마비

## 본 프로젝트 적용 가치

### 본 프로젝트의 환각 처리 현 방식

본 프로젝트는 **이미 환각을 결정론으로 회피**:
- 사주: 만세력 엔진 (engine/saju/) — 결정론 계산, LLM 무관
- 작명: name_* 모듈 8개 — 결정론 알고리즘
- 한자 풀: 8525자 Unihan JSON — 결정론 화이트리스트
- 관상 메트릭: MediaPipe 결정론, LLM은 작문만
- 안전: 43+ safety 모듈 — 결정론 게이트

→ LLM은 **작문(서술 문체 변환)만** 담당. 사실 추론은 결정론 엔진. 본 보고서가 제안하는 "LLM=번역기·포매터" 패턴을 **이미 본 프로젝트 전체에서 채택**.

### Neo4j 도입 평가

ADR-007에서 Neo4j는 "외부 보고서 누적 후 도입 검토"로 보류. 현 시점 재평가:

| 평가 항목 | 결과 |
|---|---|
| 외부 보고서 누적? | ✅ 본 시점 8건 |
| 본 프로젝트가 RAG를 쓰는가? | ❌ 사용 안 함 |
| 본 프로젝트가 LLM에 사실 추론을 시키는가? | ❌ 결정론 엔진 사용 |
| 정형화 가능한 관계망이 있는가? | 🟡 한자-획수-오행, 사주-천간지지 등 일부 있음 |
| 현 Obsidian Vault로 부족한가? | ❌ 현재 충분 (Markdown + frontmatter + 백링크) |

**판정**: **Neo4j 미도입 결정 유지**. 본 보고서는 RAG 환각 회피용으로 KG를 제안하나, 본 프로젝트는 애초에 RAG 미사용이라 KG 필요성 없음.

### 시사점 (채택 가능)

본 보고서의 구조 권장은 본 프로젝트 철학과 일치:
- "LLM은 번역기·포매터" = 본 프로젝트의 "결정론 엔진 + LLM 작문 분리" (CLAUDE.md §0)
- "스키마 강제 추출" = 본 프로젝트의 JSON schema 강제 출력 패턴
- "자가 수정 루프" = 본 프로젝트의 output_safety_gate retry 로직과 유사

## 본 시스템 반영

### ❌ 본 프로젝트 미적용 (불필요)

- Neo4j 도입 — RAG 미사용 + Obsidian Vault 충분
- NL2GQL — 본 프로젝트 사용자 쿼리는 자유 텍스트가 아닌 폼 기반 입력
- 다중 에이전트 오케스트레이션 — 본 프로젝트 현재 단일 LLM 작문 호출로 충분

### 🟡 시사점 (이미 채택된 철학의 정당화)

본 보고서가 본 프로젝트의 핵심 설계 결정(결정론 엔진 + LLM 작문 분리)을 외부 자료로 정당화. ADR-007 (Vault 선택, Neo4j 보류) 결정이 옳았음을 입증.
