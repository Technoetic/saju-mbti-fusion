## 사용법

영속화 + ADR-010 검증 후 `engine/divination/name_aesthetic.py` 신규 모듈

본 항목은 ADR-010 사실성 분리에 가장 민감 — "통계적으로 선호된다" 같은
수사가 가장 흔히 등장하는 영역.

---

## 프롬프트 본문

```
한국 작명 SaaS의 어감·발음 선호도 점수를 다음 요구사항에 맞춰
객관 데이터로 조사·정리해주세요.

## 컨텍스트

본 SaaS는 작명 추천 알고리즘에 다음 점수를 이미 사용:
- 81수리 길흉
- 자원오행 사주 보강
- 발음오행 상생
- 두음법칙 적용
- 불용한자 제외

추가하려는 어감 점수:
- 같은 점수 후보 중 "사회적으로 자주 쓰이는·발음이 자연스러운" 이름 우선

## 요구사항

### 1. 객관 데이터 출처만 (도그마 거부)
다음은 객관 데이터:
- 통계청 인구동향조사 (출생신고 인기 이름 통계)
- 국립국어원 발음·음운 공식 분석
- 학술 음운론 논문 (KCI 검색 가능)

다음은 거부:
- 작명원 주관 평가 ("이 이름은 어감이 좋다")
- "통계적으로 선호된다" 수사 (실제 표본 크기·연도 없으면 거부)
- "사주에 좋다" 등 운명론적 표현

### 2. 음운 결합 자연스러움 점수
한국어 음운론 기반:
- 받침 + 다음 글자 첫 자음 결합 시 자연스러움 (예: "선영" 받침 ㄴ + ㅇ = 자연 / "박파" ㄱ + ㅍ = 어색)
- 자음군 회피 (어색한 자음 연속)
- 모음 균형 (양성·음성 모음 배합)

출력:
```json
{
  "rule_id": "batchim_smooth",
  "description": "받침 ㄴ + 다음 자음 ㅇ/ㅎ → 자연 결합",
  "score_bonus": 0.05,
  "source": "국립국어원 표준발음법 §29"
}
```

### 3. 시대별 인기 이름 코퍼스
- 통계청 출생신고 통계 최근 10년 (2015-2024)
- 연도별 남·여 인기 100위 이름
- 음절 단위 분해 후 빈도 산출

출력:
```json
{
  "syllable": "준",
  "gender": "male",
  "frequency_rank_2024": 3,
  "frequency_count": 1234,
  "source": "통계청 인구동향 2024"
}
```

본 데이터로 추천 알고리즘이 "최근 자주 쓰이는 음절" 가점 부여.

### 4. 거부 영역 (절대 미포함)
- "X 이름은 어감이 나쁘다 / 흉하다" 인과 평가
- "이 발음은 운이 막힌다" 운명 표현
- "사주에 따라 어감이 다르다" 식 학파 도그마
- 출처 없는 "사회적 선호" 주장

### 5. 출처 검증 (ADR-010 사실성 분리 의무)
- 통계청 자료: 정확한 보고서명 + 발간 연도 + URL
- 국립국어원 자료: 표준어·표준발음법 §번호 명시
- 학술 논문: KCI URL 또는 DOI + 저자·발간 연도·학회지명

마스킹된 출처 ("XX대 학위논문" 등)는 거부.

### 6. 회귀 테스트 데이터
음절 결합 50쌍 + 자연스러움 판정 + 사유:

```json
[
  {"name": "선영", "score": 0.85, "reason": "받침 ㄴ + ㅇ 자연"},
  {"name": "박파", "score": 0.30, "reason": "ㄱ + ㅍ 자음군 회피 위반"},
  ...
]
```

본 50쌍은 본 SaaS 회귀 테스트에 그대로 사용됩니다.

## 출력 형식

마크다운 보고서 + JSON 데이터:
1. 음운 결합 규칙표 (JSON — 본 SaaS data/ 직접 적용 형식)
2. 시대별 인기 음절 빈도 (JSON)
3. 회귀 테스트 50쌍 (JSON)
4. 출처 (모두 검증 가능한 URL·서지)
5. 본 시스템 적용 시 면책 요구 사항 ("어감 점수는 추천 정렬 가점이지 예언 아님")
```
