## 컨텍스트

본 SaaS는 이미 다음을 구현했습니다:
- 만세력 계산: 천간4·지지4 도출 (자체 모듈 engine/saju/pillars.py)
- 오행 분포 카운팅: 木·火·土·金·水 개수 (단순 카운팅, 학파 무관)
- 일간 식별 (day_master)

다음을 추가하려 합니다:
- 일간 기준 강약 판정 → 용신 1~2개 도출
- 결정론 if-then 규칙 (LLM 무관)
- 추론 근거(왜 이 용신인가) 노출

## 요구사항

### 1. 학파 비교 (Trade-off 분석)
다음 4개 학파의 차이를 객관적으로 비교:
- 억부론(抑扶論): 자평진전(子平眞詮, 청대 심효첨)
- 조후론(調候論): 궁통보감(窮通寶鑑, 청대)
- 격국론(格局論): 연해자평(淵海子平, 명대 서승)
- 통관론(通關論): 적천수(滴天髓, 청대 임철초)

각 학파별:
- 핵심 원리 (한 문단)
- 알고리즘화 난이도 (low/medium/high) + 사유
- 결과의 안정성 (동일 사주가 항상 같은 용신 도출되는가)
- 한국 작명 시장 점유율 (출처 명시 — 작명원 통계·논문 등)

### 2. 채택 권장 1개 + 사유
4개 중 본 시스템(결정론 SaaS)에 가장 적합한 학파 1개를 권장. 사유는:
- 알고리즘 명료성 (규칙이 명시적인가)
- 외부 변수 의존도 (절기·시진·지장간 등이 필요한가)
- "동일 입력 → 동일 출력" 결정론 보장 가능 여부

### 3. 채택 학파의 결정론 규칙 명세
선택한 학파의 용신 도출을 Python pseudo-code 수준으로 명세:
- 입력: pillars dict (천간4·지지4·일간)
- 출력: yongshin (1~2개 오행) + 추론 근거 텍스트

예시:
```python
def derive_yongshin(pillars):
    day_master = pillars['day_master']
    # 단계 1: 일간 강약 판정
    if ... : strength = "strong"
    # 단계 2: 조후 판정 (월령 기준)
    season = month_to_season(pillars['month'])
    # 단계 3: 용신 도출
    if strength == "strong" and season == "summer": yongshin = "水"
    ...
    return {"yongshin": yongshin, "reason": "..."}
```

### 4. 검증 데이터셋 (필수)
실제 사주 50건 + 각 사주의 도출된 용신 + 추론 근거.
형식: JSON 배열
```json
[
  {
    "pillars": {"year": "庚午", "month": "壬午", "day": "乙巳", "hour": "癸未"},
    "day_master": "乙",
    "derived_yongshin": ["水"],
    "reason": "일간 乙(목)이 한여름(午月) 화왕절에 태어나 신약. 수(水)로 화기를 식히고 일간을 생함.",
    "school_source": "조후론·궁통보감"
  },
  ...
]
```

이 50건은 본 SaaS 회귀 테스트에 그대로 사용됩니다.

### 5. 출처·반증 가능성 (ADR-010 사실성 분리 의무)
다음을 명시:
- 1차 문헌 출처 (책 제목·저자·시대) — ISBN 마스킹 금지
- 2차 인용 (한국학술지·논문) — KCI 검색 가능한 정확한 서지
- 학파 내 이견 (같은 학파 안에서도 해석 차이 있다면 명시)

### 6. 거부할 내용 (포함 금지)
다음은 본 시스템 정책상 절대 거부 — 보고서에 포함 시 본문화 X:
- "이 사주는 단명한다 / 큰 부자가 된다" 같은 인과 예언
- "통계적으로 유의미하다" 같은 수사 (실제 p-value·표본 크기 없으면 거부)
- 의료·법률·금융 자문 표현
- 익명·비공식 출처에 의존한 학파별 빈도 비교
