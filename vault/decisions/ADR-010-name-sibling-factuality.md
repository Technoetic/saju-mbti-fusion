# ADR-010: 가족 서열 한자 모듈 — 사실성 분리 원칙

## 배경

2026-05-17 외부 딥리서치 보고서 수신 (`한국 정통 성명학의 서열별 상대적
불용한자 판정 알고리즘 및 데이터베이스 구축 심층 보고.md`).

최초 AI 검토에서 보고서를 무비판적으로 "⭐⭐⭐⭐⭐ 즉시 본문화 가능"으로
영속화함. 사용자 지적:

> "옵시디언으로 무조건 구조하지말고 잘못된 것은 수정하면서 진행하는 거자?
>  팩트가 아닌건 지식이 아니잖아?"

→ AI 환각·아첨(sycophancy)이 vault에 그대로 누적되면 vault가 지식이 아니라
**검증되지 않은 권위의 더미**가 됨. 본 ADR로 사실성 분리 원칙을 영속화.

## 원칙

본 시스템(사주·작명·관상 SaaS)에 채택되는 모든 외부 보고서·문헌·주장은
다음 3 등급으로 분리되어야 한다:

| 등급 | 의미 | 처리 |
|---|---|---|
| 🟢 **팩트** | 자전·문헌·관습으로 개별 검증 가능 | 채택 |
| 🟡 **구조** | 검증 불가하나 시스템 설계 명제로 채택 | 객관 라벨로 채택 |
| 🔴 **도그마** | 검증 불가 + 위해 가능 (의료·법률 인과 주장) | **폐기** |

## 본 보고서 적용 결과

### 🟢 팩트 (채택)
- 한자 어휘 의미 (자전 검증)
- 백중숙계 호칭 체계 (의례·예기 출전) → `references/baekjungsukgye.md`
- 학파 명칭 실존 (단 보고서가 작명 학파로 호명한 것은 부정확)
- 현대 가정 형태 (외동·재혼·입양·쌍둥이) 사실

### 🟡 구조 (시스템 명제로 채택)
- 4 카테고리 분류 (firstborn/lastborn/middleborn/only_child)
- 3단계 부적합도 (STRONG/WEAK/CONDITIONAL — "흉화 강도"가 아님)
- Gender 변수 (호칭 사실 매칭)
- 5 현대 가정 예외 처리 로직
- 충돌 우선순위 규칙

### 🔴 도그마 (폐기)
- 흉화 물리 발현 (단명·관재·신체 훼손·이혼) → ADR-006 충돌
- "통계적 유의성" 수사 (출처·표본 0)
- 학파별 빈도 정량 비교 (출처 0)
- 우주적 기운·파동·역행 (형이상학 명제)
- 마스킹된 출판 서지 (검증 불가)

## 채택

**B 채택 (사실성 분리 후 본문화)**.

### 구체 변경

1. **모듈명**: `name_sibling_bulyong` → `name_sibling_preference`
   - "불용(不用)"은 인과 주장 → "선호(preference)"로 객관화
2. **데이터 필드**:
   - `severity`: STRONG / WEAK / CONDITIONAL (부적합도, 호칭 직접성)
   - `reason`: 어휘 의미 + 호칭 사실만 (인과 주장 삭제)
   - `sources`: "전통 관습"으로 일괄 표기 (학파 출전 검증 보류)
3. **사용자 출력 표현**:
   - ✅ "전통 관념상 ○○ 서열에 더 자주 사용되는 글자입니다."
   - ✅ "○○ 서열에서는 사용 빈도가 낮은 글자입니다."
   - ❌ "이 글자를 사용하면 [흉화 결과]가 발생합니다."
   - 모든 출력에 면책: "참고용이며 의료·법률 자문이 아닙니다."

## 검토한 옵션

### A. 보고서 전면 폐기
- 장점: 가장 보수적
- 단점: 한자 어휘·호칭 구조 등 검증 가능한 부분도 함께 버림

### B. 사실성 분리 채택 (채택)
- 장점: 검증 가능한 부분만 채택, 도그마 폐기
- 단점: 검토 비용 (한자 128자 reason 객관화 작업)

### C. 무비판 채택 (기각)
- 1차 AI가 시도했던 방식 — vault 신뢰성 파괴
