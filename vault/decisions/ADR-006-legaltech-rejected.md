# ADR-006: 보고서 §6 리걸테크 영역 거부 (법원 신청서·CRM)

## 배경

작명 보고서 §6는 다음을 권고:
1. 개명 허가 신청서 자동 생성 (사유서 텍스트 자동 기입)
2. 1개월 신고 기한 CRM 알림 (푸시·카카오톡)
3. 필수 서류 4종 안내

## 검토

### §6.1 신청서 자동 생성 — ❌ 거부
- 변호사법 §109 (법률사무 취급) 위반 우려
- "단명 등을 암시하는 불용한자" 같은 흉화 문구를 법원 문서에 자동 기입은 책임 소재 모호
- 본 시스템 `legal_notice` 모듈이 "법률 자문 거절" 명시 — 직접 충돌

### §6.2 1개월 신고 기한 CRM — ❌ 거부
- 법적 의무 사항을 앱이 책임지는 구조
- 알림 누락 시 사용자가 과태료 부과 — 책임 소재 모호
- GDPR·KR PIPA 동의 별도 필요

### §6.3 필수 서류 안내 — 🟡 조건부 가능
- "법원에 어떤 서류가 필요한지" 단순 정보 안내는 면책 가능
- 자동 생성·기입은 불가
- 추후 검토 가능

## 채택

**§6 전체 거부 유지**. 향후 변호사 협업·법무팀 검토 거쳐 §6.3 단순 안내만 별도 ADR로 검토 가능.

## 면책

- 본 시스템은 "참고용 작명 진단" 도구일 뿐
- 법원 행정 절차는 사용자 자율 또는 변호사 의뢰

## 우회 — 사용자가 따로 가능한 행동

- 변호사 사이트 안내 링크만 외부에 노출 (제휴 모델)
- "법원 신청 방법은 변호사·법무사에게 문의하시게" 안내문

## 향후 재검토 조건

- 법무 자문 비용 확보
- 변호사 협업 파트너십
- 면책·약관 강화
