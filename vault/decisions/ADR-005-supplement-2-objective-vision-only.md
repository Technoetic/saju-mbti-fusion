# ADR-005 Supplement 2 — 운학 도사 시각 객관 묘사 한정 (Opus 사전학습 운명 매핑 차단)

## 상태

Accepted (2026-05-17). ADR-005 본문(2026-05-16) + Supplement 1(2026-05-18 Phase 12 4중 신호) 위에 **Opus의 사전학습 운명 매핑 사용을 차단**하는 추가 제약을 부과.

## 맥락

Supplement 1(Phase 12)은 결정론 점수 4중 신호를 Opus에 주입했으나, 시스템 프롬프트가 **여전히 운명 해석**(초년·중년·말년 복록, 학문복·재물복·인덕)을 작성 형식으로 요구. 그 결과:

- Opus의 사전학습된 마의상법·신상전편 통설이 그대로 사용자 출력에 반영
- "이마가 넓으니 학문복이 있겠구먼" 같은 **검증 불가 인과 매핑**이 ADR-010 사실성 분리 정신 위반
- 결정론 점수는 정확하나, LLM 본문은 결국 학파 도그마

사용자 결정 (2026-05-17):

> "Opus 4.7는 시각 객관 묘사만 해야 하지 않겠니?"

운명·길흉·시간 흐름 해석은 **본 시스템에서 제거**. 5형·12궁·삼정 명칭은 **영역명**으로만 유지 (Option B).

## 결정

**운학 도사 = 시각 객관 묘사 전문가**. Opus는 다음만 수행:

| 영역 | 허용 | 금지 |
|---|---|---|
| 얼굴 윤곽 | 둥근·각진·긴·짧은 형태 묘사 | "각진 턱이라 의지 강함" 성격 매핑 |
| 이마 | 넓이·평평함·주름 결 | "이마 넓으니 초년 학문복" 시간·운 매핑 |
| 눈썹·눈 | 짙기·길이·휨, 눈빛 강도 | "눈빛 또렷하니 명궁 좋다" 운명 매핑 |
| 코 | 곧음·휨·콧방울 두께 (형태만) | "콧방울 두툼하니 재물복" 매핑 |
| 입·턱 | 두께·각도·살 정도 | "턱 두툼하니 말년 복록" 시간 매핑 |
| 기색 | 맑은·붉은·창백한·노란기 (색상만) | "붉은 기색이니 화병" 의료 추론 |
| 5형 명칭 | "토형의 결로구먼 — 가로세로 균형" 형태 인용 | "토형이라 신용 두텁다" 성격 단정 |
| 12궁 명칭 | "명궁이 또렷하다" 영역 묘사 | "명궁이 또렷하니 평생운 밝다" 운명 매핑 |
| 삼정 명칭 | "상정 영역의 형태" 영역 묘사 | "상정이 좋으니 초년운" 시간 매핑 |

## 구현

### `_FACE_SYSTEM` 절 추가/재구성

신규 절 (face_reading.py:37~107):

1. **[근본 원칙 — ADR-010 사실성 분리 절대 준수]** — 시각 객관 묘사 한정 선언
2. **[허용 — 시각 객관 묘사만]** — 9개 카테고리 명시 (윤곽·이마·눈썹·눈·코·입·턱·기색·대칭)
3. **[엄격 금지 — 운명·해석 매핑]** — 7개 금지 카테고리:
   - 시간 차원 해석 (초년/중년/말년 복록, 학문복, 재물복, 인덕)
   - 12궁 운명 매핑
   - 5형 운명 매핑
   - 점쟁이 어휘 (대박·대운·금전수·운명)
   - 단정 예언
   - 학파 직접 인용 (마의상법·신상전편)
   - 외모 평가·미추 비교
4. **[명칭 사용 — 영역명만, 운명 매핑 X]** — Option B 명문화

### `[작성 형식]` 재설계

| 이전 (Supplement 1) | 본 Supplement 2 |
|---|---|
| 초년의 지혜·중년의 기개·말년의 복록 흐름 | 전체 인상 → 이마 형태 → 중정 형태 → 하정 형태 → 또렷한 시각 특징 |
| 5단락 시간/운명 매핑 | 5단락 시각 객관 묘사 (운명 X) |

### 안전 거절 절

운명 해석을 요청받으면 단일 거절 줄:

> "허허, 이 늙은이는 그대의 얼굴 형상을 비추어 드릴 뿐, 운명의 길흉은 헤아리지 않는다네"

후 객관 묘사로 복귀.

### `_build_user_text` 안내문

마무리 지시 변경 (face_reading.py:206~214):

> "운학 도사의 어조로 **시각 객관 묘사**를 해주시게. 운명·길흉·시간 흐름 해석은 절대 금지 — 보이는 형태·색상·균형만 사극풍 어조로 풀어 전한다."

흐름: "전체 인상 → 이마(상정 영역) 형태 → 눈썹·눈·코(중정 영역) 형태 → 입·턱(하정 영역) 형태 → 그대만의 또렷한 시각 특징 한 가지 → 마무리".

## Supplement 1 (Phase 12) 와의 관계

- Supplement 1의 4중 신호 주입(palace_scores + face_shape)은 **유지** — 결정론 점수가 묘사의 정량 근거로 인용됨
- Supplement 1의 작성 형식(시간 운명 매핑 5단락)은 **본 Supplement가 supersede**
- 두 supplement는 직렬 합산: 4중 신호 + 시각 객관 묘사 한정 = 현 운영 정책

## Opus 사전학습 운명 매핑 차단 메커니즘

Opus 4.7은 마의상법·신상전편·달마상법 등의 통설을 사전학습. 시스템 프롬프트만으로는 **완벽 차단 불가** (Anthropic 모델 카드도 명시). 본 supplement의 다층 방어:

1. **시스템 프롬프트 명시 금지**: 7개 카테고리 + 직접 키워드("초년·학문복·재물복") 명시
2. **사용자 메시지 재강조**: `_build_user_text` 끝에 "운명·길흉·시간 흐름 해석은 절대 금지" 명시
3. **결정론 점수 인용 유도**: 묘사에 결정론 점수를 정량 근거로 인용하면 추상적 운명 해석으로 빠지기 어려움
4. **안전 거절 안전망**: 사용자가 명시 요청해도 거절 후 복귀

100% 차단 보장 X — Opus가 우회 표현(예: "이마의 결이 학문에 어울리는…")을 만들 가능성. 운영 모니터링 의무.

## 한계

- **Opus 사전학습 완벽 차단 불가**: 프롬프트 우회 가능성. 운영 시 출력 샘플링 + 금지 키워드 자동 후처리 필터 검토
- **사극풍 어조 vs 객관 묘사 균형**: 운학 도사 어조 유지하되 추상 운명 해석 자제 — 미세 보정 필요 시 추가 supplement
- **사용자 기대 갭**: "관상 봐주세요" → 운명 풀이 기대. "운명의 길흉은 헤아리지 않는다네" 응답 시 사용자 만족도 저하 가능 → UI/UX에서 사전 안내 검토 (🔵 사업 단계)
- 본 ADR은 **immutable**. 추후 정책 변경 시 새 supplement
