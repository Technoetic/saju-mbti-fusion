# §5 어감·선호도 데이터 (작명 추천 시 어감 가점)

## 무엇

작명 추천 시 "어감이 좋다 / 발음이 부드럽다" 같은 사회적 선호도를
결정론 점수로 반영. 본 보고서 §5.

## 왜

현재 작명 추천은 81수리·자원오행·발음오행·두음법칙·불용한자만 사용.
"같은 점수의 후보 중 사회적으로 선호되는 어감"을 우선순위에 반영 불가.

## 차단 사유

선호도는 **사회 데이터**라 본 프로젝트 vault에서 도출 불가:
- 시대별 인기 이름 (10년 단위로 변동)
- 음운 결합 선호 (예: "민준" vs "민준희" — 어느 게 더 선호되는가?)
- 발음 어색함 (예: 받침 + 다음 글자 첫 자음 조합)

## 필요한 외부 입력

본 보고서가 받은 도그마 문제(가짜 인용·"통계적 유의성" 수사)를 회피하기
위해 다음 명시 요청:

- 출처 검증 가능: 통계청 인기 이름·국립국어원 음운 분석 등 공식 데이터
- "통계적으로 선호된다"가 아닌 "최근 N년 출생신고 인기 1000위" 형식
- 학파·도그마 무관 (작명원 주관 평가 X)

→ 딥리서치 프롬프트는 본 노트의 ## 딥리서치 프롬프트 절 참조.
